# Утренняя рутина перед запуском к пользователям

Запускается раз в день перед началом работы. ~30 секунд.

## Команда

```bash
cd /c/Users/Анна/Documents/project/sotka
python scripts/morning_check.py
```

## Что проверяет

| Блок | Что и почему |
|---|---|
| **Статусы лотов** | Active >5000 (есть товар) + Active с протухшим окном <5% (статусы синхронны) |
| **Карта** | Покрытие координатами >=60% (UX карты не ломается) |
| **Скоринг** | Active со score >=90% + score>=80 >=100 (есть что показывать в премиуме) |
| **Дисконты** | Распределение лотов по дисконту — есть ли товар под УТП «−80%» |
| **Назначение** | По всем категориям (ИЖС, ЛПХ, СНТ, сельхоз, коммерция, промка) — нет ли провала классификатора |
| **HTML-страницы** | Главная, каталог, тарифы, блог, кабинет, админка, сравнение — все 200 OK |
| **Свежесть данных** | Самый свежий лот опубликован <6 ч назад (скрейпер живой) |
| **Heatmap** | 85 регионов в карте плотности + ≥60 со средним дисконтом |

## Цвета сигналов

- ✓ Зелёный — норма, не трогать
- ⚠ Жёлтый — на грани, через день-два упадёт в красный, держать в уме
- ✗ Красный — действовать сегодня, без этого к пользователю не ходим

## Что делать на каждый красный сигнал

### `update_lot_statuses не отрабатывает`
Проверить, что Celery beat и worker запущены, посмотреть логи последнего часа:
```bash
ssh root@5.35.93.163 "cd /opt/zemlya && docker compose logs --since 1h beat worker | grep statuses"
```
Если задача не запускается — рестарт: `docker compose restart beat worker`.

### `enrich_with_rosreestr отстаёт`
Если backlog растёт быстрее, чем разгребается, увеличить `batch_size` или добавить второй воркер:
```bash
docker compose exec worker celery -A worker.celery_app call tasks.scrape_tasks.enrich_with_rosreestr --kwargs '{"batch_size":3000}'
```

### `Категория 'X' слишком пустая`
Открыть `backend/services/scraper_torgi.py`, в `PURPOSE_KEYWORDS` добавить новые ключевые слова. Прогнать переразметку:
```bash
docker compose exec worker python -c "from backend.services.rubrics import reclassify_all; reclassify_all()"
```

### `Свежие лоты не приходят`
torgi.gov мог поменять API или перестал пускать прокси. Проверить:
```bash
docker compose exec api curl -s 'http://prox:8000' -I
docker compose logs --since 30m worker | grep torgi
```

### `Страница X вернула 5xx`
Frontend SSR упал — проверить логи:
```bash
docker compose logs --since 30m web
```

### `Свежие лоты не приходят`
Если задача `scrape_torgi_gov` падает, посмотреть последний вызов:
```bash
docker compose logs --since 4h worker | grep -i 'torgi.gov\|scrape_torgi'
```

## Когда запускать

- Утром перед работой — обязательно
- Перед публикацией поста в TG-канал — обязательно (нельзя звать людей на сломанный сервис)
- После любого деплоя — обязательно (через 5 минут после рестарта)
- После любой важной правки скрейпера/скоринга — через час, чтобы новые задачи отработали

## Расширения скрипта на будущее

Когда подключишь:
- **Sentry/Bugsnag** — добавить проверку количества ошибок за 24ч (>10 → красный)
- **Платежи** — добавить проверку «успешных платежей за 24ч» (если платежи начались, не должно быть нулевых дней)
- **Telegram-бот** — пинг вебхука, проверка что обновления от users.telegram.org приходят
- **Email-доставка** — отправка тестового письма на твой адрес и проверка через IMAP
- **TLS** — просрочка сертификата (за 14 дней до истечения — жёлтый, за 3 — красный)
