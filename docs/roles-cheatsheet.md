# Роли агента — шпаргалка для Анны

Короткий справочник: какие «роли» есть у Claude при работе над Sotka,
что каждая делает и когда включается. Технический вариант — в `AGENTS.md`,
здесь — простыми словами.

**Главное**: тебе НЕ нужно называть роль вручную. Claude переключается сам
по типу твоего запроса. Ты вмешиваешься, только если хочешь переопределить.

---

## 7 ролей

### 🟦 Spec — постановщик задачи
**Что делает**: превращает размытую идею в чёткие сценарии и критерии «готово».
Находит крайние случаи, о которых легко забыть.
**Включается сам**, когда ты говоришь размыто: «хочу чтобы…», «добавь возможность…».
**Зачем тебе**: чтобы не получить «не то, что хотела» — сначала проговариваются детали.

### 🟩 Architect — проектировщик
**Что делает**: решает, куда встроить фичу, как назвать, нужна ли новая таблица
в БД, какой будет API. Взвешивает варианты.
**Включается сам**, когда задача трогает базу данных, новые разделы, крупную перестройку.
**Зачем тебе**: чтобы изменения ложились в систему аккуратно, а не «костылём сбоку».

### 🟨 Implementer — исполнитель
**Что делает**: пишет код по готовому плану. Маленькими порциями, по одному коммиту.
**Включается сам** для чётких мелких задач: «поправь текст», «добавь кнопку», «почини баг».
**Зачем тебе**: основная рабочая лошадка — большая часть задач идёт через него.

### 🟧 Test — тестировщик
**Что делает**: пишет автотесты, чтобы фича не сломалась в будущем. Фиксирует баги тестом.
**Включается сам** после критичной логики или когда найден баг.
**Статус**: тестовой инфраструктуры пока нет — это в планах модернизации.

### 🟥 Reviewer — ревьюер
**Что делает**: проверяет готовый код на баги, дыры в безопасности, поломки.
**Включается сам ОБЯЗАТЕЛЬНО** перед деплоем всего, что трогает платежи,
авторизацию, тарифы, webhook. Или когда ты скажешь «проверь».
**Зачем тебе**: ловит ошибки до того, как они попадут к пользователям.

### 🟪 Refactor — чистильщик
**Что делает**: убирает дубли, улучшает имена, удаляет мёртвый код. Поведение
не меняет — только делает код чище.
**Включается** по твоей просьбе: «почисти», «отрефактори».

### ⬜ Docs — документатор
**Что делает**: обновляет документацию проекта (CLAUDE.md, AGENTS.md, changelog),
когда что-то важное поменялось.
**Включается сам** при значимых изменениях.

---

## Когда говорить роль вручную

Почти никогда. Но можешь переопределить, если хочешь:

| Хочешь | Скажи |
|---|---|
| Обсудить дизайн до написания кода | «не пиши код, поработай как architect» |
| Пропустить обсуждение, сразу делать (для мелочи) | «просто сделай, как implementer» |
| Отдельный проход проверки готового кода | «прогони как reviewer» |
| Почистить код без новых фич | «отрефактори этот участок» |

## Цепочка для крупной фичи

Когда задача большая, роли работают по цепочке — автоматически:

```
Твоя идея
  → Spec     (уточняет: что именно, какие сценарии)
  → Architect (проектирует: куда встроить, схема БД, API)
  → ты одобряешь дизайн
  → Implementer (пишет код)
  → Reviewer  (проверяет перед деплоем)
  → Деплой
```

Тебе не нужно вести по шагам. Ты подключаешься в 2 точках: одобрить дизайн
и одобрить перед деплоем. Остальное агент делает сам.

Для мелочи (текст, цвет, мелкий баг) цепочка не нужна — сразу Implementer.
